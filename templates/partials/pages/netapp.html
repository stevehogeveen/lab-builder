Factory reset NetAppdestructive
This is a manual ONTAP boot-menu wipe workflow. It erases configuration and data on the selected node disks and should only be used for a lab rebuild.
What the app will do
Build a reset runbook, discover/probe USB console access, and only start a serial reset sequence after the console-specific destructive confirmation is entered.
What stays manual
Do not automate force giveback or HA takeover/giveback decisions. Reset one controller only when the serial console and HA state are understood.
Console factory reset execution
This starts a background serial worker that logs in, optionally sends the node reboot command below, waits for the ONTAP boot menu, and sends boot-menu option 4. Use it only with the console connected to the controller you intend to wipe.
Used for HA disable and default reboot command when the console is at ONTAP CLI.
Leave blank if the controller is already rebooting or already at the ONTAP boot menu.
Only option 4 is supported by this helper.
Optional. Some ADP/shared-disk reset flows ask for the partner name.
